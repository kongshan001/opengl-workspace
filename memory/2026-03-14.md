# 2026-03-14 进展日志

## Remote Shell v2.1.0 - 审计日志功能 ✅

### 新增模块

**`common/audit.py`** - 审计日志核心模块
- `AuditEventType` 枚举：定义所有审计事件类型
  - 认证：LOGIN_SUCCESS, LOGIN_FAILED, LOGOUT
  - 命令：COMMAND_EXEC, COMMAND_SUCCESS, COMMAND_FAILED
  - 文件：FILE_LIST, FILE_DOWNLOAD, FILE_UPLOAD
  - 隧道：TUNNEL_CREATE, TUNNEL_REMOVE
  - 用户：USER_CREATE, USER_DELETE, USER_PASSWORD_CHANGE
  - 系统：SERVER_START, SERVER_STOP, CLIENT_CONNECT, CLIENT_DISCONNECT
- `AuditLogger` 类：单例模式
  - 日志记录：按日期分文件存储
  - 日志轮转：10MB 自动分割，自动压缩旧日志
  - 日志清理：30 天自动清理过期日志
  - 查询功能：支持按类型、用户、IP、时间、成功/失败筛选
  - 统计功能：24h/3d/7d/30d 事件统计

### 服务端集成

**`server/server.py`**
- 客户端连接/断开记录
- 认证成功/失败记录
- 命令执行及结果记录
- 文件操作记录（列表/下载/上传）

### Web 端集成

**`web/web_server.py`**
- Web 登录/登出记录
- 用户管理操作记录
- 隧道创建/删除记录
- 新增 API：
  - `GET /api/audit/list` - 查询审计日志
  - `GET /api/audit/stats` - 获取统计数据

### Web 界面

新增"审计"标签页：
- 统计卡片：总事件、成功/失败数、活跃用户、独立 IP
- 过滤器：事件类型、用户名、IP、时间范围
- 日志列表：时间、事件类型、描述、用户、IP

### 日志存储

- 路径：`/var/log/remote-shell/audit/`
- 格式：`audit-YYYY-MM-DD.log`（JSON 行格式）
- 压缩：`audit-YYYY-MM-DD.log.gz`

---

## 技术要点

1. **线程安全**：使用 `threading.Lock` 保证并发写入安全
2. **单例模式**：`get_audit_logger()` 返回全局唯一实例
3. **JSON 行格式**：每行一条 JSON 记录，便于解析和追加
4. **自动轮转**：超过 10MB 自动分割，避免单文件过大
5. **便捷方法**：`audit_log()` 函数直接调用，无需手动获取实例
